# 0003-前端路由与权限控制方案

## 状态

已接受

## 背景

MES系统需要实现完整的权限控制体系,包括:
- 页面访问权限(用户只能访问有权限的页面)
- 按钮操作权限(用户只能使用有权限的功能按钮)
- 路由守卫(未登录用户跳转登录页,首次登录强制修改密码)

## 决策

### 1. 路由模式

**选择**: Hash模式 (`#/user/list`)

**理由**:
- 无需后端nginx配置支持
- 开发部署简单,适合快速迭代
- 后续可切换到History模式(仅需修改路由配置和nginx)

### 2. 路由分类

系统路由分为三类:

| 类型 | 说明 | 示例 |
|------|------|------|
| **公共路由** | 无需认证即可访问 | `/login`, `/403`, `/404` |
| **认证路由** | 需要登录,但无需特定权限 | `/dashboard`, `/change-password`, `/profile` |
| **权限路由** | 需要登录+特定权限 | `/user/list` (需要 `user:list` 权限) |

### 3. 路由守卫逻辑

```javascript
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token')
  const userInfo = localStorage.getItem('userInfo')
  const needChangePassword = localStorage.getItem('needChangePassword')

  // 1. 访问登录页
  if (to.path === '/login') {
    if (token) {
      next('/dashboard')  // 已登录,跳转首页
    } else {
      next()  // 未登录,显示登录页
    }
    return
  }

  // 2. 访问公共页面(403/404等)
  if (to.meta.public) {
    next()
    return
  }

  // 3. 检查登录状态
  if (!token) {
    next('/login')
    return
  }

  // 4. 检查密码状态(首次登录/密码过期)
  if (needChangePassword === 'true' && to.path !== '/change-password') {
    next('/change-password')
    return
  }

  // 5. 检查页面权限
  if (to.meta.permission) {
    const permissions = JSON.parse(userInfo).permissions || []
    if (!permissions.includes(to.meta.permission)) {
      next('/403')  // 无权限
      return
    }
  }

  // 6. 放行
  next()
})
```

### 4. 动态路由注册

权限路由采用**动态注册**模式:

```javascript
// 登录成功后,根据用户权限动态添加路由
const routes = [
  {
    path: '/user/list',
    component: () => import('@/views/user/UserList.vue'),
    meta: { permission: 'user:list' }
  },
  // ... 其他权限路由
]

// 根据用户权限过滤并注册
const userPermissions = userInfo.permissions
const allowedRoutes = routes.filter(r =>
  !r.meta.permission || userPermissions.includes(r.meta.permission)
)
router.addRoute('Layout', allowedRoutes)  // 动态添加到布局路由下
```

### 5. 按钮权限控制

采用**自定义指令**方式:

```javascript
// 注册全局指令
app.directive('permission', {
  mounted(el, binding) {
    const permission = binding.value  // 如 'user:add'
    const permissions = JSON.parse(localStorage.getItem('userInfo')).permissions

    if (!permissions.includes(permission)) {
      el.parentNode?.removeChild(el)  // 无权限,移除按钮
    }
  }
})

// 在组件中使用
<button v-permission="'user:add'">新增用户</button>
<button v-permission="'user:edit'">编辑用户</button>
```

### 6. 前端布局结构

```
Layout (布局组件)
├── Sidebar (侧边栏菜单)
│   └── 根据权限动态显示菜单项
├── Header (顶部导航)
│   ├── 用户信息
│   ├── 退出按钮
├── Main (主内容区)
│   └── <router-view> (子路由渲染区)
```

### 7. Token处理

**请求拦截器**:
```javascript
request.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})
```

**响应拦截器**:
```javascript
request.interceptors.response.use(
  response => {
    if (response.data.code !== 200) {
      message.error(response.data.message)
      return Promise.reject(response.data)
    }
    return response.data.data
  },
  error => {
    if (error.response?.status === 401) {
      // 被强制登出或Token过期
      localStorage.clear()
      router.push('/login')
      message.error(error.response.data.message || '登录已失效,请重新登录')
    }
    return Promise.reject(error)
  }
)
```

## 理由

### 为什么选择Hash模式
- 开发阶段无需nginx配置
- Vue3默认支持,配置简单
- 未来切换History模式成本低

### 为什么选择动态路由注册
- 避免用户看到无权限的菜单项
- 代码按需加载,减少首屏加载时间
- 权限变更后重新登录即可刷新路由

### 为什么使用自定义指令控制按钮权限
- 声明式语法,代码简洁
- 无需每个按钮手动判断权限
- 自动移除无权限按钮,用户无感知

## 影响

### 前端结构影响
- 需要创建 `@/router/index.js` 配置路由守卫
- 需要创建 `@/directives/permission.js` 注册权限指令
- 需要创建 `@/utils/request.js` 配置请求拦截器
- 所有API请求统一使用封装的request工具

### 后端接口影响
- 登录接口返回数据需包含用户权限列表
- 所有接口统一返回 `{ code, message, data }` 格式
- Token无效时返回 401 状态码

## 登录流程完整时序

```
用户输入工号+密码 → 前端调用登录接口
↓
后端验证 → 返回 { token, userInfo, needChangePassword }
↓
前端存储 token 和 userInfo 到 localStorage
↓
检查 needChangePassword:
  - true → 跳转 /change-password
  - false → 跳转 /dashboard
↓
动态注册权限路由 → 渲染侧边栏菜单
↓
用户操作 → 按钮根据权限显示/隐藏
↓
用户请求 → 拦截器自动携带Token
↓
后端验证Token → 正常响应 或 401(被强制登出)
↓
401响应 → 前端清除存储 → 跳转登录页 → 提示用户
```
