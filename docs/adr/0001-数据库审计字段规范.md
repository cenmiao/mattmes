# 0001-数据库审计字段规范

## 状态

已接受

## 背景

MES系统作为生产制造执行系统,涉及大量生产数据的记录和追溯。为满足审计要求和问题追溯需求,所有数据的创建和修改都需要有明确的责任人记录。

## 决策

**所有业务表必须包含四个审计字段**:

| 字段 | 类型 | 说明 |
|------|------|------|
| created_by | VARCHAR(64) | 创建人工号 |
| create_time | DATETIME | 创建时间 |
| updated_by | VARCHAR(64) | 最后修改人工号 |
| update_time | DATETIME | 最后修改时间 |

## 理由

1. **可追溯性**: 出现数据问题时,可快速定位责任人
2. **审计合规**: 满足制造行业的质量审计要求
3. **自动化填充**: 通过MyBatis-Plus的自动填充功能,这些字段可在插入和更新时自动设置,无需业务代码手动处理

## 影响

- 所有新建表必须包含这四个字段
- MyBatis-Plus需要配置MetaObjectHandler实现自动填充
- created_by 和 updated_by 存储用户工号(而非用户ID),便于直接查看
