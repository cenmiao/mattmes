# 0005-菜单管理方案

## 状态

已接受

## 背景

MES系统前端需要侧边栏菜单导航,菜单数据的管理方式影响系统的灵活性和开发效率。

## 决策

**当前阶段**: 前端代码写死菜单配置

**未来阶段**: 根据需求考虑重构为数据库存储

### 前端菜单配置结构

```javascript
// src/router/menus.js
export const menus = [
  {
    title: '系统管理',
    icon: 'setting',
    children: [
      {
        title: '用户管理',
        path: '/user/list',
        permission: 'user:list'
      },
      {
        title: '角色管理',
        path: '/role/list',
        permission: 'role:list'
      },
      {
        title: '权限管理',
        path: '/permission/list',
        permission: 'permission:list'
      },
      {
        title: '登录日志',
        path: '/login-log/list',
        permission: 'login-log:list'
      }
    ]
  },
  // 后续业务模块菜单...
]
```

### 菜单渲染逻辑

1. 前端根据用户权限过滤菜单项
2. 无权限的菜单项不显示
3. 支持多级菜单结构

## 理由

### 为什么选择前端写死
1. **MES菜单相对固定**: 生产管理系统菜单结构稳定,变动频率低
2. **减少初期开发工作量**: 无需开发菜单管理模块和后端接口
3. **快速迭代**: 前期优先实现核心业务功能

### 什么时候考虑数据库存储
1. 需要运维人员动态配置菜单
2. 多租户场景,不同租户菜单不同
3. 菜单变动频繁,无法接受重新部署

## 影响

### 前端影响
- 菜单配置集中在 `src/router/menus.js`
- 新增菜单需修改代码并重新部署

### 后端影响
- 无需开发菜单管理接口
- 登录接口无需返回菜单数据
