### 技术约束

- **权限编码格式**: 模块级如 `user`，按钮级如 `user:add`
- **权限编码唯一**: `permission_code` 必须唯一，重复返回400错误
- **父权限规则**: `parent_id=null` 表示模块级权限，有值表示按钮级子权限
- **树形结构**: 模块包含其下所有按钮权限，返回嵌套JSON
- **级联删除**: 删除模块级权限时同时删除其下所有按钮级权限
- **关联清理**: 删除权限时同步清理 `sys_role_permission` 关联数据
- **响应格式**: 使用已有的 `Result` 和 `ResultCode`
- **异常处理**: 使用 `BusinessException` 抛出业务异常

### 关键实现点

1. **创建权限**:
   - 验证 `permission_code` 唯一性（已存在返回400）
   - 设置 `permission_type`: 模块级=1，按钮级=2
   - 模块级权限 `parent_id=null`（或0），按钮级权限 `parent_id` 指向父模块

2. **编辑权限**:
   - 只允许修改 `permission_name`、`description`、`sort_order`
   - `permission_code`、`permission_type`、`parent_id` 不可修改

3. **删除权限**:
   - 如果是模块级权限（parent_id=null/0），先查询并删除所有子权限
   - 删除 `sys_role_permission` 关联数据
   - 删除 `sys_permission` 记录

4. **查询列表**:
   - 返回所有权限，按 `sort_order` 排序
   - 包含字段：id, permissionName, permissionCode, permissionType, parentId, description, createTime

5. **查询树形结构**:
   - 查询所有模块级权限（permission_type=1）
   - 为每个模块查询其下按钮级权限（parent_id=模块id）
   - 构建嵌套结构：`{ id, permissionName, permissionCode, children: [...] }`

### 数据库表结构

`sys_permission` 权限表：
- id (主键自增)
- permission_name (权限名称)
- permission_code (权限编码，唯一索引)
- permission_type (权限类型: 1=模块, 2=按钮)
- parent_id (父权限ID: 0表示顶级)
- sort_order (排序号)
- description (权限描述)
- 审计字段: created_by, create_time, updated_by, update_time

`sys_role_permission` 角色权限关联表：
- id, role_id, permission_id
