   ```typescript
   // 参考已有的 auth.ts 风格
   import request from '@/utils/request'
   
   // 9个接口函数
   getUsers(params: UserQuery): Promise<PageResult<UserResponse>>
   getUser(id: number): Promise<UserResponse>
   createUser(data: UserCreateRequest): Promise<UserResponse>
   updateUser(id: number, data: UserUpdateRequest): Promise<UserResponse>
   assignRoles(id: number, roleIds: number[]): Promise<void>
   resetPassword(id: number): Promise<{ password: string }>
   disableUser(id: number, reason: string): Promise<void>
   enableUser(id: number): Promise<void>
   deleteUser(id: number): Promise<void>
   ```

2. **搜索栏**:
   - 工号/姓名输入框（`el-input`，placeholder="工号/姓名"）
   - 状态下拉（`el-select`，选项：全部/启用/禁用）
   - 角色下拉（`el-select`，选项动态加载，需调用角色列表接口 `GET /api/roles`）
   - 搜索按钮 + 重置按钮

3. **数据表格** (`el-table`):
   - 工号列：等宽字体（IBMMono / IBM Plex Mono），主色 `#0EA5E9`，可点击查看详情

   - 角色列：`el-tag` 标签组（多个角色显示多个Tag）

     - admin账号（user_no=admin）Switch禁用不可切换
     - 禁用状态下hover Switch时 `el-tooltip` 显示禁用原因
   - 最后登录列：等宽字体，灰色；无登录时显示"从未登录"
   - 操作列：图标按钮组（编辑、重置密码、删除）
     - admin账号删除按钮禁用（灰色不可点击）

4. **新增用户弹窗** (`el-dialog` + `el-form`):
   - 工号输入（必填）
   - 姓名输入（必填）
   - 手机号输入（可选）
   - 邮箱输入（可选）
   - 初始密码输入（必填，默认生成或手动输入）
   - 角色选择（`el-select` 多选，动态加载角色列表）

5. **编辑用户弹窗**:

   - 角色选择可编辑

   - 提示"新密码已生成，请告知用户"

7. **权限指令控制**:
   - 新增按钮：`v-permission="user:add"`
   - 编辑按钮：`v-permission="user:edit"`
   - 重置密码按钮：`v-permission="user:reset-password"`
   - 禁用/启用Switch：`v-permission="user:disable"`

### 权限编码注意

路由中注册的权限是 `user:read`（控制页面访问），按钮级权限使用 `user:add`、`user:edit`、`user:reset-password`、`user:disable`、`user:delete`。两者不冲突，分别控制不同粒度。

### 角色列表接口

角色下拉需要动态加载角色列表。后端 `RoleController` 应已实现 `GET /api/roles` 接口（#07已完成）。如该接口不可用，可暂时硬编码角色选项。
