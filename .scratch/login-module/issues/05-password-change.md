# 密码修改功能

## Parent

参考 `.scratch/login-module/PRD.md`

## What to build

实现密码修改端到端功能，支持首次登录强制修改、密码过期修改、用户主动修改三种场景。

核心功能：
- 密码强度校验（至少8位、必须包含字母+数字）
- 旧密码验证（非首次登录场景）
- 新密码与旧密码不能相同
- 修改成功后Token立即失效（强制重新登录）
- 更新password_update_time为当前时间
- 清空login_error_count（重置错误计数）

三种场景：
1. 首次登录强制修改（无需旧密码验证）
2. 密码过期修改（需旧密码验证）
3. 用户主动修改（需旧密码验证，从用户下拉菜单入口）

## Agent Brief

### 任务摘要

实现密码修改端到端功能，支持首次登录强制修改、密码过期修改、用户主动修改三种场景。

### 实现范围

**创建文件**:
1. `mattmes-common/src/main/java/com/matt/mes/common/utils/PasswordUtils.java` — 密码强度校验工具类
2. `mattmes-system/src/main/java/com/matt/mes/system/service/ChangePasswordService.java` — 密码修改服务接口
3. `mattmes-system/src/main/java/com/matt/mes/system/service/impl/ChangePasswordServiceImpl.java` — 密码修改服务实现
4. `mattmes-system/src/main/java/com/matt/mes/system/dto/ChangePasswordRequest.java` — 请求DTO
5. `mattmes-web/src/main/java/com/matt/mes/controller/ChangePasswordController.java` — 密码修改接口
6. 单元测试文件（ChangePasswordServiceTest.java, PasswordUtilsTest.java）

**修改文件**:
- 无需修改现有文件

### 技术约束

- **密码加密**: 使用已有的 `BCryptPasswordEncoder` (Spring Security)
- **Token失效**: 调用已有的 `TokenStorage.invalidateToken(userId)` 使Token失效
- **响应格式**: 使用已有的 `Result` 和 `ResultCode`
- **异常处理**: 使用已有的 `BusinessException`

### 三种场景处理

1. **首次登录强制修改**:
   - 前端传入 `isFirstLogin=true`
   - 无需验证旧密码（`oldPassword` 可为空）
   - 修改成功后更新 `password_update_time` 为当前时间

2. **密码过期修改**:
   - 前端传入 `isFirstLogin=false`
   - 必须验证旧密码
   - 修改成功后更新 `password_update_time` 为当前时间

3. **用户主动修改**:
   - 前端传入 `isFirstLogin=false`
   - 必须验证旧密码
   - 从用户下拉菜单入口触发

### 密码强度校验规则

`PasswordUtils.validateStrength(String password)` 返回 `List<String>` 不满足的原因：

1. **长度不足**: `password.length() < 8` → 返回 "密码长度至少8位"
2. **缺少字母**: 无字母字符 → 返回 "密码必须包含至少1个字母"
3. **缺少数字**: 无数字字符 → 返回 "密码必须包含至少1个数字"
4. **校验通过**: 返回空列表

### 关键实现点

1. **ChangePasswordService 核心流程**:
   ```
   1. 校验 newPassword 与 confirmPassword 是否一致
   2. 校验 newPassword 与 oldPassword 是否相同（非首次登录场景）
   3. 调用 PasswordUtils.validateStrength() 校验密码强度
   4. 非首次登录场景：验证旧密码是否正确
   5. 更新用户密码（BCrypt加密）
   6. 更新 password_update_time = 当前时间
   7. 清空 login_error_count = 0
   8. 调用 tokenStorage.invalidateToken(userId) 使Token失效
   9. 返回成功消息
   ```

2. **ChangePasswordController 接口**:
   - POST `/api/change-password`
   - 请求体: `{oldPassword?: string, newPassword: string, confirmPassword: string, isFirstLogin: boolean}`
   - 成功响应: `{code: 200, message: "密码修改成功，请重新登录"}`
   - 失败响应: `{code: 400, message: 具体错误原因}`

3. **错误消息映射**:
   - 密码强度不足 → 400 + 具体原因（如"密码长度至少8位"）
   - 旧密码错误 → 400 + "旧密码错误"
   - 新旧密码相同 → 400 + "新密码不能与旧密码相同"
   - 两次密码不一致 → 400 + "两次密码不一致"
