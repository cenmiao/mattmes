# 用户管理CRUD

## Parent

参考 `.scratch/login-module/PRD.md`

## What to build

实现用户管理完整CRUD功能，包含用户创建、编辑、角色分配、状态切换、密码重置、删除、查询筛选。

核心功能：
- 创建用户（工号、姓名、初始密码、角色分配）
- 编辑用户（姓名、手机号、邮箱）
- 为用户分配多个角色（权限取并集）
- 重置密码为默认密码（如Reset@123）
- 禁用用户（记录禁用原因ADMIN_MANUAL）
- 启用用户（解除禁用状态）
- 删除用户（清理账号数据）
- 30天未登录自动禁用（定时任务或登录时检查）
- 查询用户列表（搜索、筛选）
- 保护超级管理员账号（不可删除、不可禁用）

### 技术约束

- **密码加密**: 使用 `BCryptPasswordEncoder` (已在 `SecurityConfig` 中配置)
- **默认密码**: `Reset@123`
- **超级管理员保护**: `user_no=admin` 的用户不可删除、不可禁用
- **响应格式**: 使用已有的 `Result` 和 `ResultCode`
- **异常处理**: 使用 `BusinessException` 抛出业务异常

### 关键实现点

1. **创建用户**:
   - 验证工号唯一性（已存在返回400）
   - 初始密码BCrypt加密存储
   - `password_update_time` 设为 `null`（首次登录强制修改）
   - 插入 `sys_user_role` 关联表（如有角色分配）

2. **编辑用户**:
   - 不可修改工号
   - 只允许修改 `name`、`phone`、`email`

3. **分配角色**:
   - 先删除旧的 `sys_user_role` 关联
   - 再插入新的关联记录

4. **重置密码**:
   - 重置为 `Reset@123`
   - BCrypt加密存储
   - 返回明文密码给前端显示

5. **禁用用户**:
   - 检查是否为超级管理员（`user_no=admin`），禁止禁用返回400
   - 设置 `status=0`, `disable_reason=ADMIN_MANUAL`
   - 清空 `current_token`（强制登出）

6. **启用用户**:
   - 设置 `status=1`
   - 清空 `disable_reason`

7. **删除用户**:
   - 检查是否为超级管理员（`user_no=admin`），禁止删除返回400
   - 使用 MyBatis-Plus 逻辑删除（`@TableLogic` 已在实体类中配置）

8. **查询用户列表**:
   - 支持参数：`userNo`（模糊搜索）、`name`（模糊搜索）、`status`（精确筛选）、`roleId`（精确筛选）
   - 返回字段包含：最后登录时间、当前状态、角色列表
   - 使用 MyBatis-Plus 分页插件

9. **定时任务（可选）**:
   - 检查30天未登录用户自动禁用（`disable_reason=AUTO_INACTIVE`）
   - 或在登录时检查并禁用（已在 #04 中实现）
