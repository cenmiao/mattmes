# 端到端验证

## Parent

参考 `.scratch/login-module/PRD.md`

## What to build

手动验证完整登录模块端到端流程，确保所有功能正常工作，满足用户故事验收标准。

核心验证流程：
- admin首次登录并强制修改密码
- 登录后菜单正常显示
- 权限控制正常生效
- 并发登录测试
- 各管理页面功能验证

## Acceptance criteria

- [ ] **前置准备**：
  - 执行数据库Schema脚本(#1)
  - 执行系统初始化脚本(#18)
  - 启动后端服务(mattmes-web)
  - 启动前端服务(mattmes-ui)
- [ ] **首次登录流程验证**：
  - 使用admin/Admin@123登录
  - 系统提示首次登录需修改密码
  - 跳转修改密码页面
  - 设置新密码（如Admin@456）
  - 提交成功跳转登录页
  - 使用新密码重新登录成功
- [ ] **仪表盘和菜单验证**：
  - 登录成功跳转仪表盘
  - Sidebar显示所有菜单项（用户、角色、权限、日志）
  - Header显示用户名和工号
  - 用户下拉菜单正常（修改密码、退出）
- [ ] **权限控制验证**：
  - 创建测试用户testuser，分配部分权限
  - 使用testuser登录
  - 菜单仅显示有权限的项
  - 无权限菜单隐藏
  - 尝试直接访问无权限路由跳转403
  - 无权限按钮隐藏
- [ ] **并发登录验证**：
  - 在浏览器A使用admin登录
  - 在浏览器B使用admin登录
  - 浏览器B显示冲突弹窗（上次登录时间、IP）
  - 浏览器B点击确认登录成功
  - 浏览器A请求时提示"账号已在其他设备登录"
  - 浏览器A跳转登录页
- [ ] **密码错误锁定验证**：
  - 创建测试账号
  - 连续输入错误密码5次
  - 显示锁定提示和剩余时间
  - 等待锁定结束或手动解锁
- [ ] **用户管理页面验证**：
  - 列表显示正常
  - 搜索筛选正常
  - 新增用户成功
  - 编辑用户成功
  - 分配角色成功
  - 重置密码成功
  - 禁用/启用用户正常
  - admin账号不可删除/禁用
- [ ] **角色管理页面验证**：
  - 列表显示正常
  - 新增角色成功
  - 分配权限成功
  - SUPER_ADMIN角色不可删除
- [ ] **权限管理页面验证**：
  - 树形结构显示正常
  - 新增模块级权限成功
  - 新增按钮级权限成功
  - 删除权限成功
- [ ] **登录日志页面验证**：
  - 列表显示所有登录记录
  - 搜索筛选正常
  - 导出Excel成功下载
  - 登出类型显示正确（手动/强制/过期）
- [ ] **退出登录验证**：
  - 点击退出登录
  - 清空localStorage
  - 跳转登录页
  - 登录日志记录登出类型=MANUAL
- [ ] **30天未登录自动禁用验证**（模拟测试）：
  - 修改某用户last_login_time为30天前
  - 该用户登录时提示账号已禁用
  - 管理员启用后可正常登录
