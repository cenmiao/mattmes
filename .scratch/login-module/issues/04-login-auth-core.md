# 登录认证核心流程

## Parent

参考 `.scratch/login-module/PRD.md`

## What to build

实现完整的登录认证端到端流程，从用户提交工号密码到Token生成返回，包含所有安全机制。

核心功能：
- 工号+密码验证（BCrypt校验）
- JWT Token生成（Payload: userId, userNo, exp, iat，有效期6小时）
- Token数据库存储（DatabaseTokenStorage方案）
- 登录错误计数（连续5次错误锁定10分钟，显示剩余锁定时间）
- 并发登录检测（检测已有在线会话，返回409冲突提示）
- 强制登录处理（确认后使旧Token失效）
- 被强制登出检测（拦截器检测Token是否匹配当前用户，返回401）
- 首次登录强制修改密码检测（返回needChangePassword标志）
- 30天未登录自动禁用（登录时检查，禁用原因AUTO_INACTIVE）
- 超级管理员跳过权限验证（后续拦截器使用）

端到端流程：
1. 前端POST /api/login {userNo, password, forceLogin?: boolean}
2. 后端验证 → 生成Token → 存储到sys_user.current_token → 返回Token+用户信息
3. 前端存储Token到localStorage
4. 后续请求携带Token → 拦截器验证 → 业务处理

### 技术约束

- **密码加密**: 使用 `BCryptPasswordEncoder` (Spring Security)
- **JWT 库**: `io.jsonwebtoken:jjwt-api:0.12.5`（已在 pom.xml 中）
- **Token 有效期**: 6小时
- **锁定策略**: 连续5次错误锁定10分钟
- **响应格式**: 使用已有的 `Result` 和 `ResultCode`

### 关键实现点

1. **登录验证流程**:
   - 检查用户是否存在 → 检查账号状态 → 检查锁定状态 → 验证密码 → 生成Token → 更新登录信息
   - 密码错误时递增 `login_error_count`，达到5次设置 `lock_time`
   - 锁定期间返回400 + 剩余锁定时间（毫秒）

2. **并发登录处理**:
   - 检查 `current_token` 是否有效（未过期）
   - 有效则返回409 + 当前登录时间/IP
   - `forceLogin=true` 时清空旧用户 Token

3. **Token 拦截器**:
   - 从 Header 获取 Token → 解析验证 → 检查与数据库是否匹配
   - 不匹配返回401 + "账号已在其他设备登录"

4. **首次登录/密码过期检测**:
   - `password_update_time == null` → 首次登录
   - `password_update_time` 超过30天 → 密码过期
   - 返回 `needChangePassword=true`

5. **30天未登录禁用**:
   - 检查 `last_login_time`，超过30天设置 `status=0`, `disable_reason=AUTO_INACTIVE`
